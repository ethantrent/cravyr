  <action>
Create `.github/workflows/eas-build.yml`:

```yaml
name: EAS Build
on:
  push:
    branches: [main]
    paths:
      - 'apps/mobile/**'
      - 'packages/shared/**'

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - uses: pnpm/action-setup@v4
        with:
          version: 9
      - run: pnpm install --frozen-lockfile
      - uses: expo/expo-github-action@v8
        with:
          eas-version: latest
          token: ${{ secrets.EXPO_TOKEN }}
      - run: eas build --platform all --profile production --non-interactive
        working-directory: apps/mobile
```

Environment secrets required:
- `EXPO_TOKEN` — from expo.dev account settings

The workflow only triggers on changes to `apps/mobile/**` or `packages/shared/**` to avoid unnecessary builds on API-only changes.
  </action>
</task>

3. **Location Permission String**: ✅ `NSLocationWhenInUseUsageDescription` set to specific text: "Cravyr uses your location to find restaurants near you and show distance information." (not generic)

4. **Minimum Functionality**: ✅ App has: onboarding flow, swipe deck, restaurant detail with photos/hours/directions, saved list management, user preferences, settings screen, account deletion

5. **Privacy Policy URL**: ❓ Needs a privacy policy page hosted somewhere (Vercel landing page or a simple static page). Required for App Store submission.

6. **App Icon**: ❓ `app.config.ts` has `icon` commented out — needs a 1024x1024 app icon

7. **Screenshots**: ❓ App Store requires screenshots for each device size — capture after final UI polish
