<objective>
Create the push notification infrastructure: a `push_tokens` table in Supabase, a token registration API endpoint, and mobile-side token registration that fires on every authenticated foreground event.

Purpose: SC-2 requires push tokens upserted on every authenticated app open with no stale accumulation. This plan delivers the data layer and registration pipeline so Plan 05-02 can focus purely on sending notifications.

Output: Token registration pipeline live end-to-end; push_tokens table populated on app open.
</objective>

Create `supabase/migrations/20260415100000_push_tokens.sql`:

```sql
CREATE TABLE public.push_tokens (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  expo_push_token text NOT NULL,
  platform text NOT NULL CHECK (platform IN ('ios', 'android')),
  updated_at timestamptz DEFAULT now(),
  UNIQUE(user_id, platform)
);

ALTER TABLE public.push_tokens ENABLE ROW LEVEL SECURITY;

CREATE POLICY push_tokens_owner ON public.push_tokens
  FOR ALL
  USING (user_id = (SELECT auth.uid()))
  WITH CHECK (user_id = (SELECT auth.uid()));
```

The `UNIQUE(user_id, platform)` constraint ensures one token per user per device platform — re-registration upserts rather than accumulating stale rows.

Create `apps/api/src/routes/notifications.ts` following the existing pattern from swipes.ts:

```typescript
import { Router } from 'express';
import type { Request, Response } from 'express';
import { createClient } from '@supabase/supabase-js';
import { requireAuth } from '../middleware/auth';

const supabaseAdmin = createClient(
  process.env.SUPABASE_URL!,
  process.env.SUPABASE_SERVICE_ROLE_KEY!,
);

export const notificationsRouter = Router();
notificationsRouter.use(requireAuth);

/**
 * POST /api/v1/notifications/register
 * Body: { expo_push_token: string, platform: 'ios' | 'android' }
 *
 * Upserts the Expo push token for the authenticated user.
 * UNIQUE(user_id, platform) ensures no stale token accumulation.
 */
notificationsRouter.post('/register', async (req: Request & { user?: { id: string } }, res: Response) => {
  const userId = req.user?.id;
  if (!userId) { res.status(401).json({ error: 'Unauthorized' }); return; }

  const { expo_push_token, platform } = req.body;
  if (!expo_push_token || typeof expo_push_token !== 'string') {
    res.status(400).json({ error: 'expo_push_token is required' }); return;
  }
  if (!['ios', 'android'].includes(platform)) {
    res.status(400).json({ error: 'platform must be ios or android' }); return;
  }

  const { data, error } = await supabaseAdmin
    .from('push_tokens')
    .upsert(
      { user_id: userId, expo_push_token, platform, updated_at: new Date().toISOString() },
      { onConflict: 'user_id,platform' },
    )
    .select('id')
    .single();

  if (error) { res.status(500).json({ error: error.message }); return; }
  res.status(201).json(data);
});
```

Mount in `apps/api/src/server.ts`:
```typescript
import { notificationsRouter } from './routes/notifications';
app.use('/api/v1/notifications', notificationsRouter);
```

**Step 3**: In `_layout.tsx`, add push token registration after session is established.

Use a soft-prompt pattern: do NOT request permission on first launch. Request permission only when the user has completed onboarding and is on the tabs group.

On every authenticated foreground (AppState 'active' or initial mount with session):
1. `Notifications.getPermissionsAsync()` — if already granted, get token + upsert
2. If not granted yet, show nothing — the soft prompt comes from a separate UI later
3. `Notifications.getExpoPushTokenAsync({ projectId })` — get the token
4. POST to `/api/v1/notifications/register` with the token + Platform.OS

The registration is fire-and-forget — failures do not block the app.
