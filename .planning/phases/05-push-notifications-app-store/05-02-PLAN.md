<objective>
Build the push notification sending service and the daily 6PM "Tonight's Picks" reminder cron job.

Purpose: SC-1 requires a test user to receive a push notification at ~6PM their local time. This plan delivers the sending pipeline and the cron scheduling logic. Since Render free/starter tier has no native cron, we use a simple setInterval-based approach that fires hourly and checks which timezone buckets should receive notifications.

Output: Daily 6PM reminder notifications live; invalid token cleanup working.
</objective>

Create `apps/api/src/services/push.ts`:

The service wraps `expo-server-sdk`:
1. `sendPushNotifications(tokens: string[], title: string, body: string)` — chunks tokens into batches of 100 (Expo limit), sends via `expo.sendPushNotificationsAsync`, returns ticket IDs
2. `checkReceipts(ticketIds: string[])` — calls `expo.getPushNotificationReceiptsAsync`, identifies `DeviceNotRegistered` errors, returns invalid tokens
3. `cleanupInvalidTokens(invalidTokens: string[])` — deletes matching rows from `push_tokens` table

Token validation before sending: use `Expo.isExpoPushToken(token)` to filter out malformed tokens.

Handle chunking per Expo docs: max 100 notifications per request.

Create `apps/api/src/services/cron.ts`:

**Approach**: Since we don't have per-user timezone data in the MVP, use a simplified approach:
1. `setInterval` fires every hour
2. On each tick, query all users who have at least one save from the last 24 hours AND have a push token
3. Send a notification: "You have picks waiting! 🍽️ Check your Tonight's Picks before they close."
4. Track last notification time per user in a `notification_log` or simple in-memory Map to avoid spamming (skip users notified in the last 20 hours)

**For MVP simplicity:** Use a single daily notification window. The cron fires at the top of each hour; when UTC hour matches a configured "send hour" (default 23:00 UTC ≈ 6PM ET), it sends to all eligible users. This can be upgraded to per-user timezone logic post-MVP.

Export `startCronJobs()` and call from `server.ts` after `app.listen()`.

Schedule receipt checking 15 minutes after each send batch to clean up invalid tokens.
