# Phase 4 — UI Design Contract

> Visual and interaction contract for Phase 4: Swipe Core + Secondary Screens.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — React Native (not applicable for shadcn) |
| Preset | not applicable |
| Component library | React Native core + expo-linear-gradient + ReanimatedSwipeable |
| Icon library | @expo/vector-icons (Ionicons set) |
| Font | System default (San Francisco on iOS, Roboto on Android via RN default) |

**Rationale:** This is a React Native / Expo project. shadcn and Tailwind are web-only tools and are not applicable. All styling uses React Native StyleSheet objects with the tokens defined in this contract. Typography uses the OS system font stack, which renders natively at the appropriate weight fidelity on each platform.

---

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, badge inner padding, divider margin |
| sm | 8px | Compact row padding, metadata gap between chips |
| md | 16px | Default horizontal screen padding, card inner padding, list row vertical padding |
| lg | 24px | Section gaps within detail sheet, action button group spacing |
| xl | 32px | Card stack bottom clearance above action buttons, empty state vertical centering offset |
| 2xl | 48px | Tab bar clearance (safe area), top of detail view header inset |
| 3xl | 64px | Empty state illustration bottom margin |

Exceptions:
- Touch targets for all action buttons (X, Heart, Star, Undo) must be minimum **44px × 44px** (Apple HIG minimum, App Store accessibility requirement). Visual button size may be smaller; use padding to achieve touch target.
- Swipe card border radius: **16px** (not on the spacing scale — visual property).
- Picks list row thumbnail: **64px × 64px** square (not on spacing scale — fixed dimension).

| Skeleton base | `#2c2c2e` | — | Loading skeleton shimmer base |
| Skeleton highlight | `#3a3a3c` | — | Loading skeleton shimmer highlight |

**60/30/10 split:**
- 60% dominant: `#0f0f0f` — full-screen backgrounds, the majority of every screen
- 30% secondary: `#1c1c1e` — sheet surfaces, list rows, settings panels
- 10% accent: `#f97316` — primary CTA buttons only (empty state CTA, Preferences save button, tab active state). Not used on destructive actions, not used on overlay labels (those use semantic colors).

**Accent reserved for:** Empty state "Go to Preferences" button, Preferences screen "Save" button, active tab indicator. These are the only three elements that use `#f97316`.

**Semantic color contract:**
- `#22c55e` = save/positive only
- `#ef4444` = skip/destructive/delete only
- `#eab308` = superlike/premium only
- `#f97316` = primary CTA only

## Component Specifications

### Swipe Card

```
Layout:      Full-bleed card, flex:1, borderRadius 16px, overflow hidden
Photo:       expo-image, contentFit:"cover", fills card entirely
Gradient:    expo-linear-gradient, position absolute, bottom 0, left 0, right 0
             height 160px, colors: ['transparent', 'rgba(0,0,0,0.85)']
             padding 16px (md token)
Name text:   22px / weight 700 / white / bottom of gradient
Metadata:    15px / weight 400 / white opacity 0.85
             Format: "{price_level_emoji} · {primary_cuisine} · {distance}km"
             Price level emoji: $ = "•", $$ = "••", $$$ = "$$$", $$$$ = "$$$$"
Stack visual: Cards 2 and 3 visible behind top card, scale 0.96 / 0.92, translateY 8px / 16px
```

### Overlay Labels (SAVE / SKIP / SUPERLIKE)

```
Container:   borderWidth 3, borderRadius 8, padding 8px (sm token)
Text:        28px / weight 800 / matching border color
SAVE:        borderColor #22c55e, color #22c55e, rotated -15 degrees (top-left placement)
SKIP:        borderColor #ef4444, color #ef4444, rotated 15 degrees (top-right placement)
SUPERLIKE:   borderColor #eab308, color #eab308, no rotation (top-center placement)
Appearance:  Opacity interpolated from 0 at 0px drag to 1.0 at 80px drag offset
```

### Action Buttons (below deck)


## Animation Specifications

All animations must execute on the native UI thread via Reanimated worklets. No JS thread animation is acceptable.

### Swipe Card Spring (rn-swiper-list swipeRightSpringConfig / swipeLeftSpringConfig / swipeTopSpringConfig)

```
damping:   18
stiffness: 120
mass:      1
```

This produces a responsive swipe that completes in ~350ms without overshoot. If the card feels too snappy during testing, lower stiffness to 100. If it feels sluggish, increase to 140. These values are the default starting point — tune interactively.

### Overlay Label Interpolation

```
Trigger:    Card drag offset in X axis (or Y for superlike)
Opacity:    0 at 0px offset → 1.0 at 80px offset (linear interpolation)
Scale:      1.0 at 0px → 1.05 at 80px (subtle grow)
Rotation:   SAVE label: -15deg fixed. SKIP label: +15deg fixed. SUPERLIKE: 0deg.
```

### Card Stack Visual (cards 2 and 3 behind top card)

```
Card 2 (index 1): scale 0.96, translateY +8px
Card 3 (index 2): scale 0.92, translateY +16px
Transition:       withSpring({ damping: 20, stiffness: 150 }) when top card is removed
```

### Action Button Press Feedback

```
Type:       scale animation via Animated.spring (or Reanimated withSpring)
Press in:   scale 0.92, duration 80ms
Press out:  scale 1.0, spring damping 15, stiffness 200
```

### Skeleton Shimmer

```
Type:       Reanimated withRepeat(withTiming(...))
Duration:   1200ms per cycle
Easing:     Easing.inOut(Easing.ease)
Direction:  Reverse true (back-and-forth shimmer)
Value:      opacity oscillates 0.4 → 1.0 on highlight overlay
```

### Swipe-Back (Undo) Animation

```
Handled by: rn-swiper-list swiperRef.current.swipeBack()
Spring:     Library default (matches swipe spring config)
Gate:       Only call swipeBack() when undoStack.length > 0
```

## Interaction States

### Swipe Card — States

| State | Visual |
|-------|--------|
| Idle (top card) | Full-bleed photo + gradient overlay + name/meta |
| Dragging right | SAVE overlay fades in, card rotates clockwise (interpolated from drag offset) |
| Dragging left | SKIP overlay fades in, card rotates counter-clockwise |
| Dragging up | SUPERLIKE overlay fades in, no rotation |
| Released (threshold met) | Card flies off in swipe direction, next card springs to front |
| Released (threshold not met) | Card springs back to center |
| Loading | Skeleton shimmer in card shape |
| Error | Error state full-screen replaces deck |

Swipe threshold: library default (approximately 30% of card width).

### Action Buttons — States

| Button | Idle | Pressed | Disabled |
|--------|------|---------|----------|
| Skip (X) | #1c1c1e bg, #ef4444 icon | scale 0.92 | opacity 0.4 (deck empty) |
| Save (Heart) | #1c1c1e bg, #22c55e icon | scale 0.92 | opacity 0.4 (deck empty) |
| Superlike (Star) | #1c1c1e bg, #eab308 icon | scale 0.92 | opacity 0.4 (deck empty) |
| Undo | #1c1c1e bg, #ababab icon | scale 0.92 | opacity 0.4 when undoStack empty |

### Tonight's Picks — States

| State | Visual |
|-------|--------|
| Populated | FlatList of RestaurantRow items |
| Empty (no saves yet) | Centered: Ionicons "bookmark-outline" 64px #2c2c2e + "No picks yet." 16px #ababab + "Swipe right on restaurants to save them." 14px #636366 |
| Swipe in progress | Delete button revealed at right, row translates left |
| Delete confirmed | Row animates out (height collapses via Reanimated LayoutAnimation) |
| Loading | 3 skeleton rows (64px height each, shimmer) |

### Detail View Photo Gallery — States

| State | Visual |
|-------|--------|
| 1 photo | Single image, no dots |
| 2-5 photos | Dot indicators shown, swipe-to-navigate |
| Loading | expo-image blurhash placeholder (thumbhash from server) |
| Failed to load | expo-image fallback: gray #1c1c1e with Ionicons "image-outline" 32px #636366 centered |

## Accessibility Contract

All interactive elements must satisfy App Store accessibility requirements:

| Element | Requirement |
|---------|-------------|
| Action buttons (X, Heart, Star, Undo) | accessibilityLabel set: "Skip restaurant", "Save restaurant", "Superlike restaurant", "Undo last swipe" |
| Swipe card | accessibilityLabel: "{restaurant name}, {cuisine}, {distance}" |
| Overlay labels | aria-hidden (decorative — action buttons provide accessible equivalent) |
| Tab bar items | accessibilityLabel matches tab label |
| Delete Account confirm button | accessibilityRole: "button", destructive hint in accessibilityHint |
| Touch targets | Minimum 44×44pt on all interactive elements |
| Text contrast | Primary text #ffffff on #0f0f0f = 21:1 (passes AAA). Secondary #ababab on #0f0f0f = 7.3:1 (passes AA). |
