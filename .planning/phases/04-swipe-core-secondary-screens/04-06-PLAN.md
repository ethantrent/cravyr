Create `apps/api/src/routes/swipes.ts` with a POST handler that records a swipe to the Supabase `swipes` table.

The handler must:
1. Require auth via `requireAuth` middleware (mounted on the router, same as usersRouter pattern)
2. Extract `restaurant_id` and `direction` from `req.body`
3. Validate `direction` is one of the three allowed values: `'left'`, `'right'`, `'superlike'`. Return 400 `{ error: 'Invalid direction' }` if not.
4. Extract `userId` from `req.user.id` (from the verified JWT — NEVER from req.body)
5. Insert into the `swipes` table via Supabase, using a service-role client to bypass RLS on insert (same pattern as users.ts uses `adminSupabase`). The insert upserts on conflict `(user_id, restaurant_id)` so re-swiping updates rather than errors.
6. Return 201 `{ id: record.id, direction }` on success.
7. Return 500 `{ error: error.message }` on Supabase error.

```typescript
import { Router } from 'express';
import type { Request, Response } from 'express';
import { createClient } from '@supabase/supabase-js';
import { requireAuth } from '../middleware/auth';

const supabaseAdmin = createClient(
  process.env.SUPABASE_URL!,
  process.env.SUPABASE_SERVICE_ROLE_KEY!
);

export const swipesRouter = Router();

swipesRouter.use(requireAuth);

const VALID_DIRECTIONS = ['left', 'right', 'superlike'] as const;
type SwipeDirection = typeof VALID_DIRECTIONS[number];

/**
 * POST /api/v1/swipes
 *
 * Records a swipe direction for the authenticated user on a restaurant.
 * Body: { restaurant_id: string, direction: 'left' | 'right' | 'superlike' }
 *
 * SECURITY: userId comes from the verified JWT (req.user.id), never from req.body.
 * The DB trigger on swipes table auto-inserts into saves on right/superlike — Tonight's Picks
 * is populated automatically without a second API call.
 *
 * Upserts on conflict (user_id, restaurant_id) so re-swiping the same restaurant updates
 * rather than violating the unique constraint.
 */
swipesRouter.post('/', async (req: Request & { user?: { id: string } }, res: Response) => {
  const userId = req.user?.id;
  if (!userId) {
    res.status(401).json({ error: 'Unauthorized' });
    return;
  }

  const { restaurant_id, direction } = req.body as {
    restaurant_id: string;
    direction: SwipeDirection;
  };

  if (!restaurant_id || typeof restaurant_id !== 'string') {
    res.status(400).json({ error: 'restaurant_id is required' });
    return;
  }

  if (!VALID_DIRECTIONS.includes(direction)) {
    res.status(400).json({ error: 'Invalid direction. Must be left, right, or superlike.' });
    return;
  }

  const { data, error } = await supabaseAdmin
    .from('swipes')
    .upsert(
      { user_id: userId, restaurant_id, direction, swiped_at: new Date().toISOString() },
      { onConflict: 'user_id,restaurant_id' }
    )
    .select('id, direction')
    .single();

  if (error) {
    res.status(500).json({ error: error.message });
    return;
  }

  res.status(201).json(data);
});
```

**apps/api/src/routes/restaurants.ts** — GET /:id handler

The `restaurants` table in Supabase stores `photo_urls` as an array of Google Places photo reference strings (not full expiring URLs). The handler must:
1. No auth required — restaurant detail is public data (no `requireAuth`).
2. Accept `:id` as a Supabase UUID (not Google Place ID).
3. Query Supabase `restaurants` table by `id`. Return 404 `{ error: 'Not found' }` if no row.
4. Return the restaurant row as-is from Supabase. The `photo_urls` column contains photo reference strings; the mobile client renders them via Google Places photo URL pattern. (Actual URL regeneration via Google Places API is a Phase 2 concern — Phase 4 gap closure just needs the row data to flow.)
5. Cache individual restaurant detail for 2 hours using a simple in-memory Map (no external dependency). Key: `restaurant:${id}`, value: `{ data, expiresAt: Date.now() + 7200000 }`.

```typescript
import { Router } from 'express';
import type { Request, Response } from 'express';
import { createClient } from '@supabase/supabase-js';

const supabase = createClient(
  process.env.SUPABASE_URL!,
  process.env.SUPABASE_ANON_KEY!
);

// Simple in-memory cache: 2-hour TTL per CLAUDE.md guidance for individual restaurant details
const restaurantCache = new Map<string, { data: unknown; expiresAt: number }>();

export const restaurantsRouter = Router();

/**
 * GET /api/v1/restaurants/:id
 *
 * Returns a restaurant row from Supabase by its UUID.
 * photo_urls contains Google Places photo reference strings — the mobile client
 * hotlinks them per Google Places ToS (no downloading to own storage).
 * Results cached in-memory for 2 hours (node-cache equivalent per CLAUDE.md).
 */
restaurantsRouter.get('/:id', async (req: Request, res: Response) => {
  const { id } = req.params;

  // Check cache first
  const cached = restaurantCache.get(`restaurant:${id}`);
  if (cached && cached.expiresAt > Date.now()) {
    res.json(cached.data);
    return;
  }

  const { data, error } = await supabase
    .from('restaurants')
    .select('*')
    .eq('id', id)
    .single();

  if (error || !data) {
    res.status(404).json({ error: 'Restaurant not found' });
    return;
  }

  // Cache for 2 hours
  restaurantCache.set(`restaurant:${id}`, { data, expiresAt: Date.now() + 7_200_000 });

  res.json(data);
});
```

**apps/api/src/routes/recommendations.ts** — GET / handler

The handler must:
1. Require auth via `requireAuth` (recommendations are user-specific — different deck per user).
2. Read `lat` and `lng` from query params (`req.query.lat`, `req.query.lng`). Parse both as floats. Return 400 `{ error: 'lat and lng query parameters are required' }` if missing or not valid numbers.
3. Call Supabase RPC `get_restaurant_recommendations` with params `{ p_user_id: userId, p_lat: lat, p_lng: lng }`.
4. Return the scored array as JSON. Return 500 on RPC error.

```typescript
import { Router } from 'express';
import type { Request, Response } from 'express';
import { createClient } from '@supabase/supabase-js';
import { requireAuth } from '../middleware/auth';

const supabase = createClient(
  process.env.SUPABASE_URL!,
  process.env.SUPABASE_ANON_KEY!
);

export const recommendationsRouter = Router();

recommendationsRouter.use(requireAuth);

/**
 * GET /api/v1/recommendations?lat=XX&lng=YY
 *
 * Calls the Supabase PostGIS RPC function `get_restaurant_recommendations`
 * which scores nearby restaurants by distance (40%), cuisine match (25%),
 * rating (20%), and price match (15%), excluding already-swiped restaurants.
 *
 * Returns scored restaurant array ordered by rank descending.
 */
recommendationsRouter.get('/', async (req: Request & { user?: { id: string } }, res: Response) => {
  const userId = req.user?.id;
  if (!userId) {
    res.status(401).json({ error: 'Unauthorized' });
    return;
  }

  const lat = parseFloat(req.query.lat as string);
  const lng = parseFloat(req.query.lng as string);

  if (isNaN(lat) || isNaN(lng)) {
    res.status(400).json({ error: 'lat and lng query parameters are required and must be numbers' });
    return;
  }

  const { data, error } = await supabase.rpc('get_restaurant_recommendations', {
    p_user_id: userId,
    p_lat: lat,
    p_lng: lng,
  });

  if (error) {
    res.status(500).json({ error: error.message });
    return;
  }

  res.json(data ?? []);
});
```

**Step 1: Create apps/api/src/routes/saves.ts**

The handler must:
1. Require auth via `requireAuth`.
2. DELETE /:id — where `:id` is the `saves.id` UUID (the save record ID, not the restaurant ID).
3. Delete from the `saves` table WHERE `id = :id` AND `user_id = userId`. The `user_id` condition is mandatory to prevent one user deleting another user's save (ownership check at query level).
4. Use a service-role client so the DELETE bypasses RLS (consistent with users.ts pattern).
5. Return 204 on success. Return 404 `{ error: 'Save not found' }` if no rows were deleted (count === 0).

```typescript
import { Router } from 'express';
import type { Request, Response } from 'express';
import { createClient } from '@supabase/supabase-js';
import { requireAuth } from '../middleware/auth';

const supabaseAdmin = createClient(
  process.env.SUPABASE_URL!,
  process.env.SUPABASE_SERVICE_ROLE_KEY!
);

export const savesRouter = Router();

savesRouter.use(requireAuth);

/**
 * DELETE /api/v1/saves/:id
 *
 * Removes a save record (Tonight's Picks entry) for the authenticated user.
 * The WHERE clause filters on BOTH id AND user_id — prevents a user from
 * deleting another user's save even if they know the UUID.
 *
 * Called from saved.tsx after optimistic delete — failure is non-fatal
 * since the UI already removed the item from the local Zustand store.
 */
savesRouter.delete('/:id', async (req: Request & { user?: { id: string } }, res: Response) => {
  const userId = req.user?.id;
  if (!userId) {
    res.status(401).json({ error: 'Unauthorized' });
    return;
  }

  const { id } = req.params;

  const { error, count } = await supabaseAdmin
    .from('saves')
    .delete({ count: 'exact' })
    .eq('id', id)
    .eq('user_id', userId);

  if (error) {
    res.status(500).json({ error: error.message });
    return;
  }

  if (count === 0) {
    res.status(404).json({ error: 'Save not found' });
    return;
  }

  res.status(204).send();
});
```
