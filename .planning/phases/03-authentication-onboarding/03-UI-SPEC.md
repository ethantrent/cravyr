# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 3: Authentication + Onboarding.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — React Native (shadcn is web-only, not applicable) |
| Preset | not applicable |
| Component library | React Native core + @expo/vector-icons (Ionicons) |
| Icon library | @expo/vector-icons (Ionicons set) — matches Phase 4 |
| Font | System default (San Francisco on iOS, Roboto on Android) |

**Rationale:** This is a React Native / Expo project. All styling uses React Native `StyleSheet` objects with the tokens defined in this contract, carried over from Phase 4. The dark theme (`#0f0f0f` / `#1c1c1e` / `#f97316`) is already live in `app/(tabs)/_layout.tsx` — onboarding screens must match it exactly.

## Spacing Scale

Inherited from Phase 4 (source: `04-UI-SPEC.md`). No new tokens introduced.

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Icon-to-text gaps, dot indicator gap, chip inner vertical padding delta |
| sm | 8px | Preference chip paddingVertical, step progress dot gap, metadata spacing |
| md | 16px | Default horizontal screen padding, input field padding, button horizontal padding |
| lg | 24px | Section gaps, heading-to-body margin, button-to-divider spacing |
| xl | 32px | Screen top safe-area inset padding, button marginTop in auth screen |
| 2xl | 48px | Progress step row top margin, auth screen top clearance |
| 3xl | 64px | Location soft-prompt icon size, illustration margin |

Exceptions:
- Touch targets for all tappable elements (buttons, chips, social sign-in buttons) must be minimum **44px × 44px** (Apple HIG). Visual height may be less; use paddingVertical to reach 44px.
- **20px** — input field paddingVertical (gives 44px total height with 16px body text: 20×2 + 4px baseline = 44px). Falls between md (16) and lg (24); it is an input-specific height rule, not a general spacing value.
- **12px** — space between text inputs (email / password) within the auth form. Between sm (8) and md (16); it is a form-specific gutter.
- **56px** — height of the primary CTA button ("Allow Location", "Continue", "Create Account", "Sign In"). Matches Phase 4's action button diameter; an affordance-first dimension that satisfies 44px minimum with visual weight.
- **40px** — height of social sign-in buttons (Google, Apple). Matches native platform convention and provides a distinct visual size tier below the primary CTA.

## Typography

Inherited from Phase 4 (source: `04-UI-SPEC.md`). Phase 3 uses the same 4 sizes and 2 weights. No new typographic values.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|

**Declared sizes (4 values):** 13, 16, 22, 28.
**Declared weights (2 values):** 400 (regular), 700 (bold).

## Color

Inherited from Phase 4 (source: `04-UI-SPEC.md`). Full palette carried forward without modification.

| Role | Hex | Usage in this phase |
|------|-----|---------------------|
| Dominant surface | `#0f0f0f` | All onboarding screen backgrounds, auth screen background |
| Secondary surface | `#1c1c1e` | Text input backgrounds, social sign-in button backgrounds, preference chip unselected state |
| Tertiary surface | `#2c2c2e` | Divider line between email form and social buttons, input border color (unselected), inactive progress step dots |
| Primary text | `#ffffff` | All headings, input text, button labels |
| Secondary text | `#ababab` | Body/explanation copy, placeholder text, caption text, input label when unfocused |
| Accent / primary action | `#f97316` | Primary CTA buttons only (see reserved list below), active progress step dot, selected cuisine chip border + tint, active price/distance segment border + tint, focused input border |
| Error / destructive | `#ef4444` | Inline form validation error messages, failed auth error banner text |
| Apple Sign-In | `#000000` | Apple's official button background (required by Apple HIG) |
| Google Sign-In | `#ffffff` | Google's official button background with `#1c1c1e` outer container (use system approach) |

**60/30/10 split:**
- 60% dominant: `#0f0f0f` — full-screen onboarding backgrounds
- 30% secondary: `#1c1c1e` — form inputs, chips, social buttons
- 10% accent: `#f97316` — primary CTAs, active selection indicators

**Accent reserved for:**
1. Primary CTA buttons ("Allow Location", "Continue", "Create Account", "Sign In")
2. Active/selected preference chips (border + 15% opacity background tint)
3. Active price level and distance segment (same chip treatment)
4. Active step progress dot in multi-step onboarding
5. Focused text input border (1px)

Accent is **not** used on social sign-in buttons, dividers, secondary text, or error states.

**Semantic color contract (matches Phase 4):**
- `#f97316` = primary CTA only
- `#ef4444` = error / validation failure only
- No green or gold semantic colors are introduced in this phase

## Screen Flow

```
app/onboarding/
  location.tsx       — Location soft-prompt (step 0)
  location-denied.tsx — Location denied fallback (conditional)
  cuisines.tsx       — Preference step 1
  price.tsx          — Preference step 2
  distance.tsx       — Preference step 3
  auth.tsx           — Sign in / create account (final step)

Navigation:
  Each step uses router.push() to advance (stack navigation preserves back)
  Exception: location-denied.tsx replaces location.tsx in the stack (router.replace)
  Auth screen: after successful auth + preferences flush → router.replace('/(tabs)')
  Back navigation: allowed from cuisines → location, price → cuisines, distance → price
  Back navigation: NOT available from auth screen (router.replace from distance, no back to preferences mid-auth)
```

## Interaction States

### Location Soft-Prompt

| State | Visual |
|-------|--------|
| Idle | Icon + heading + body + "Allow Location" button (full opacity) |
| Tapped "Allow Location" | Button shows ActivityIndicator (16px, white) replacing label text; native dialog fires |
| Permission granted | router.push to cuisines.tsx |
| Permission denied | router.replace to location-denied.tsx |

### Location Denied Fallback

| State | Visual |
|-------|--------|
| Idle | Gray location icon + heading + body + instruction + "Open Settings" button |
| Tapped "Open Settings" | Linking.openSettings() fires — no visual change on screen |
| User returns (AppState change) | Re-check permissions; if granted now → router.replace to cuisines.tsx |

### Preference Chips (all 3 steps)

| State | Visual |
|-------|--------|
| Unselected | `#1c1c1e` bg, `#2c2c2e` border, `#ababab` text |
| Selected | `rgba(249,115,22,0.15)` bg, `#f97316` border, `#f97316` text weight 700 |
| Press (haptic) | Scale 0.96 via withSpring (damping 20, stiffness 200), immediate, native thread |

### "Continue" / Primary CTA Button

| State | Visual |
|-------|--------|
| Enabled | backgroundColor `#f97316`, text white weight 700 |
| Disabled | backgroundColor `#2c2c2e`, text `#636366` weight 400 |
| Pressed | Scale 0.97 via withSpring (damping 15, stiffness 200) |
| Loading (auth submission) | backgroundColor `#f97316`, ActivityIndicator (white, 20px) replaces text |

### Text Inputs (Auth Screen)

| State | Visual |
|-------|--------|
| Unfocused, empty | border `#2c2c2e`, placeholder text `#ababab` |
| Focused | border `#f97316` (1px) |
| Filled, valid | border `#2c2c2e` (returns to default), text `#ffffff` |
| Error | border `#ef4444`, error banner appears below CTA |

### Toggle (Sign In / Create Account)

| State | Visual |
|-------|--------|
| Sign In active | "Sign In" tab: borderBottom 2px `#f97316`, text white 700 |
| Create Account active | "Create Account" tab: same treatment |
| Inactive tab | text `#ababab` weight 400, no bottom border |

## Animation Specifications

All animations run on the native UI thread via Reanimated worklets.

### Chip Press Feedback

```
Type:    Reanimated withSpring scale
Press:   scale → 0.96
Release: scale → 1.0 (spring: damping 20, stiffness 200)
```

### Screen Transition (Onboarding Steps)

```
Type:    Expo Router default Stack.Screen slide (right-to-left push, left-to-right back)
No custom animation needed — system default matches the expected flow direction
```

### Social Sign-In Loading Overlay

```
Type:    Opacity fade-in
Enter:   opacity 0 → 1 over 150ms (withTiming, Easing.out(Easing.ease))
Exit:    opacity 1 → 0 over 100ms (withTiming)
```

### "Continue" Button Disabled → Enabled

```
Type:    backgroundColor interpolation
         Reanimated useAnimatedStyle with withTiming
Duration: 200ms
Easing:  linear
```

## Accessibility Contract

| Element | Requirement |
|---------|-------------|
| All CTA buttons | accessibilityRole: "button", minimum 44×44pt touch target |
| Text inputs | accessibilityLabel matching field name; accessibilityHint for password input: "Enter your password" |
