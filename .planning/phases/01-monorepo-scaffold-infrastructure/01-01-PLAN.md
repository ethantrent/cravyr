   - `CREATE EXTENSION postgis` (in extensions schema)

   - `CREATE TABLE public.user_preferences`
   - The `handle_swipe_save` trigger function and trigger

   - All `ALTER TABLE ... ENABLE ROW LEVEL SECURITY` statements
   - All `CREATE POLICY` statements (using `(SELECT auth.uid())` optimization pattern)

   If the generated file is thousands of lines with internal schema references, it needs aggressive cleaning. If it is a few hundred lines with only public schema objects, minimal cleanup is needed.

5. Per D-04: Do NOT add any seed data. The migration file is schema-only.
