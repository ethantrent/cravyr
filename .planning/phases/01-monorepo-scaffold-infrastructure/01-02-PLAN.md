   Add these imports at the top of the file (after the existing imports):

   import cors from 'cors';

   ```typescript
   // Security headers — disable CSP in development for Expo DevClient compatibility (Pitfall 4)
   app.use(helmet({
     contentSecurityPolicy: process.env.NODE_ENV === 'production',
   }));

   // CORS — allow requests from mobile app (per D-10)
   app.use(cors({
     origin: process.env.CORS_ORIGIN || '*',
     methods: ['GET', 'POST', 'PUT', 'DELETE'],
   }));

   // Rate limiting — 100 requests per 15 minutes per IP (per D-10)
   app.use(rateLimit({
     windowMs: 15 * 60 * 1000,
     limit: 100,
     standardHeaders: 'draft-8',
     legacyHeaders: false,
   }));

   // Body parsing
   app.use(express.json());
   ```

   Keep the health check endpoint and all route mounting UNCHANGED after the middleware stack.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client (mobile app) -> API | Untrusted HTTP requests cross this boundary; CORS and rate-limiting apply |
| API -> Supabase | Server-side requests using service role key |
| .env files -> Git repo | Secrets must never cross into version control |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | Information Disclosure | HTTP responses | mitigate | helmet middleware sets X-Content-Type-Options, X-Frame-Options, Strict-Transport-Security, removes X-Powered-By header |
| T-01-05 | Spoofing | CORS origin | mitigate | cors middleware with configurable CORS_ORIGIN env var; defaults to '*' in dev, should be restricted in production via env var |
| T-01-06 | Denial of Service | API endpoints | mitigate | express-rate-limit at 100 requests/15 min per IP with draft-8 standard headers |
| T-01-07 | Information Disclosure | .env.example files | accept | Files contain only placeholder values ("your-anon-key"), not real secrets. .gitignore excludes actual .env files. |
| T-01-08 | Information Disclosure | render.yaml | accept | Contains env var key names but not values (sync: false). Key names are not secrets. |
| T-01-09 | Elevation of Privilege | SUPABASE_SERVICE_ROLE_KEY in render.yaml | mitigate | Key is declared with `sync: false` so it must be set in Render dashboard manually, never in the YAML file itself |
</threat_model>
