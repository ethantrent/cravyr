# Quick Task: Fix res.locals.user bug in social routes + integration tests

## Bug

requireAuth (apps/api/src/middleware/auth.ts:31) sets `req.user`. The five older
routes read `req.user?.id`. The June social routes (connections.ts ×4 handlers,
matches.ts ×1) read `res.locals.user.id` instead — `res.locals.user` is undefined,
so every authenticated request throws TypeError and returns 500. Never caught
because (a) the only tests were unauthenticated 401 checks, and (b) Render deploys
were failing since June 28, so the routes were never exercised in production.

## Tasks

1. Replace `res.locals.user.id` with the codebase-standard
   `req.user?.id` + 401 guard pattern (mirrors users.ts) in both route files.
2. Add `social-endpoints.test.ts`: mock `@supabase/supabase-js` at module level
   (vi.hoisted state), cover happy paths, 401s, invalid/expired code (404),
   self-link (400), duplicate connection (409), body validation (400),
   and matches RPC parameter passing.
3. Run full test suite; atomic commit.
