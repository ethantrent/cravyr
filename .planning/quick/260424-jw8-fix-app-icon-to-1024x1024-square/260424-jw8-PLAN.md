<objective>
Apple App Store submission is blocked because `apps/mobile/assets/icon.png` is 1376×768 (landscape) instead of the required 1024×1024 square PNG with no alpha. `adaptive-icon.png` and `splash.png` share the same wrong dimensions. `splash-icon.png` is a 1×1 placeholder stub. No higher-resolution source asset (SVG, Figma, PSD, AI) exists anywhere in the repo outside `node_modules`.

This plan produces an Apple-compliant 1024×1024 icon **only after the user decides how to source it**, because a blind 768→1024 upscale of a cropped landscape would ship an ugly App Store icon for Cravyr. The plan:

1. **Audits** the current assets and generates three candidate 1024×1024 previews (center crop, left-third, right-third of the 1376×768 original) so the user has concrete visuals to reject or approve.
2. **Checkpoints** the user with a decision: supply a real source asset, approve one of the three candidates as a stopgap, or defer until a designer delivers artwork. If deferred, the plan STOPS and the App Store blocker is recorded.
3. **Generates** the final 1024×1024 `icon.png` (no alpha), 1024×1024 `adaptive-icon.png`, and a 2048×2048 `splash.png` from the approved source, via a reproducible Sharp-based script committed to `apps/mobile/scripts/generate-app-icons.mjs`.

Purpose: unblock App Store submission (Human Action #6 in STATE.md) without silently shipping a lossy icon.
Output: a square, non-alpha 1024×1024 `icon.png` the user has explicitly approved, plus a reproducible regeneration script.
</objective>

## Trust Boundaries

|----------|-------------|
| user-supplied source path (option-a) | Task 3 reads an arbitrary absolute path from the user's disk. Sharp is well-hardened but the script should not `eval` or shell-expand the path. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-260424-01 | Tampering | icon.png with sneaky alpha | mitigate | Task 3 script re-reads output via `sharp().metadata()` and fails with exit 1 if `hasAlpha === true` or `channels !== 3`. Automated verify uses `file(1)` regex `8-bit/color RGB(?!A)` to catch RGBA. |
| T-260424-02 | Tampering | wrong dimensions slip through | mitigate | Script fails on any output not exactly 1024×1024 (icons) or 2048×2048 (splash). Plan-level automated verify independently re-checks via `file(1)`. Two-layer defense. |
| T-260424-03 | Information Disclosure | user-supplied source path (option-a) could leak via logs | accept | Script only logs filename of output files, not the source path. Low-risk: solo dev machine, no shared logs. |
| T-260424-04 | Denial of Service | malformed source crashes Sharp | mitigate | Script wraps `sharp(source)` in try/catch and prints a clear error message pointing at `_preview/AUDIT.md` for remediation guidance. |
| T-260424-05 | Repudiation | lossy icon ships without user awareness | mitigate | Task 2 is a mandatory blocking checkpoint. The user must explicitly pick option-a/b/c. No auto-approval path. AUDIT.md explicitly warns about the ~33% upscale softness. |
